# 梦幻式情绪陪伴设计

## 设计理念

鼓励语池作为系统内的**低频情绪陪伴机制**，不以行为引导或情绪调节为目标，仅在用户处于停留状态时以**非确定性方式**浮现。

## 核心特点

### 1. 低频出现机制
- **非确定性**：每次显示概率约30%
- **不定期浮现**：间隔15-45秒随机
- **首次延迟**：进入页面后5-10秒才可能出现
- **显示时长**：8-15秒后自然消散

### 2. 填充式动画

不同于传统的"显性进入动画"，采用**填充式动画**：

#### 技术实现
```typescript
opacity: 0 → 1 (2500ms, 延迟300ms)
scale: 0.92 → 1 (3000ms, 延迟200ms)  
blur: 10 → 0 (2800ms, 延迟400ms)
```

#### 视觉效果
- 文本**不是突然显示**，而是伴随：
  - **遮罩**渐进变化
  - **透明度**缓慢提升
  - **模糊度**逐步清晰
- 如同**光雾在空间中缓慢凝聚**
- 将语言**轻轻托住**

#### 缓动曲线
```typescript
Easing.bezier(0.25, 0.1, 0.25, 1)
```
极其柔和的贝塞尔曲线，避免任何突兀感

### 3. 视觉设计

#### 多层次结构
1. **外层光晕**：`rgba(255, 255, 255, 0.4)`
2. **内容层**：`rgba(255, 255, 255, 0.85)` + 毛玻璃效果
3. **边框**：`rgba(255, 255, 255, 0.3)` 几乎不可见
4. **阴影**：`opacity: 0.03` 极其微弱

#### 文字样式
- 透明度：`0.75`（不强调存在感）
- 字间距：`0.5`（呼吸感）
- 行高：`24`（舒适阅读）

### 4. 语言设计

#### 调整前（评判性）
```
"今天又学到了新东西，你很棒！"
"知道得越多，对自己越温柔~"
"又进步了一点点，继续加油！"
```

#### 调整后（陪伴性）
```
"..."
"嗯"
"在呢"
"慢慢来"
"没关系的"
```

#### 设计原则
- **极简**：多用省略号和单字
- **中性**：不评价、不引导
- **存在感**：只是安静地在
- **可忽略**：不制造心理负担

## 使用场景

### 养成页面（主页）
- 怪兽安静陪伴
- 气泡**可能**出现
- 用户可以忽略
- 不影响交互

### 蛋糕奖励
- 50%概率显示文字
- 50%只显示蛋糕图标
- 3-5秒后自动消散
- 不要求用户确认

## 心理学考量

### 避免的问题
❌ 即时反馈压力（"你很棒！"）  
❌ 评判性语言（"进步了"、"加油"）  
❌ 强制关注（明显的弹窗）  
❌ 行为引导（"继续努力"）

### 实现的目标
✅ 温和的存在感  
✅ 可被忽略的陪伴  
✅ 梦幻、柔和的视觉体验  
✅ 零心理负担

## 技术细节

### 组件：DreamyBubble
```typescript
<DreamyBubble 
  text="..." 
  visible={showPhrase} 
/>
```

### 动画层次
1. **第一层**（延迟200ms）：scale变化
2. **第二层**（延迟300ms）：opacity变化
3. **第三层**（延迟400ms）：blur变化

### 消散动画
- 同样柔和的反向过程
- 1500ms缓慢消散
- 不留痕迹

## 对比：传统 vs 梦幻式

| 维度 | 传统设计 | 梦幻式设计 |
|------|---------|-----------|
| 出现频率 | 每次必现 | 30%概率 |
| 动画方式 | 滑入/弹出 | 填充式渐现 |
| 视觉强度 | 明确可见 | 半透明、可忽略 |
| 文案风格 | 鼓励、评价 | 陪伴、中性 |
| 心理负担 | 需要响应 | 可被忽略 |
| 存在时长 | 需手动关闭 | 自动消散 |
